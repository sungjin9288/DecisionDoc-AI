Use this skill when the user needs a policy or public-sector project planning brief.

Required output posture:

- Start from the decision maker's approval question.
- Separate problem, cause, intervention, operation, governance, and expected effect.
- Use restrained public-sector language.
- Do not invent statistics, institutions, budgets, contact details, certifications, or outcomes.
- Mark weak evidence as TODO rather than presenting it as confirmed.
- Keep costs, schedules, and KPI wording conditional when the source is not final.
- Include privacy, security, operating responsibility, logs, and change-management checks when AI or data systems are involved.

Draft structure:

1. Decision summary
2. Problem and policy need
3. Proposed operating model
4. Implementation path
5. Evidence status
6. Governance and risk checks
7. Open decisions
