# AI 사업 제안서 골든 예시 — 보건복지부 복지급여 AI 자동심사 시스템 구축

## 사업이해서

### 사업 배경
보건복지부는 기초생활수급, 장애인급여, 아동수당 등 15개 종류의 복지급여를 연간 840만 건 처리하고 있으나, 전체 심사 업무의 68%가 단순 서류 확인과 반복적인 데이터 대조 작업으로 구성되어 있다. 2025년 현재 복지 담당 공무원 1인당 월 처리 건수는 320건으로, OECD 평균(185건) 대비 73% 과중한 상태이다. AI 자동심사 시스템 도입을 통해 단순 업무를 자동화하고 복지 사각지대 발굴 등 고부가가치 업무에 인력을 재배치한다.

### 현황 및 문제점
- **심사 지연**: 평균 심사 소요일 21일 (긴급 생계 위기 대응 불가)
- **오심 발생**: 수작업 데이터 대조 오류로 인한 오심률 3.2% (연간 약 26만 건)
- **부정수급**: 연간 약 180억 원 규모 부정수급 발생 (중복 신청·허위 소득 신고)
- **시스템 분산**: 사회보장정보시스템, 주민등록DB, 금융정보 등 12개 시스템 수동 조회

### 사업 목표
1. 복지급여 심사 소요일 21일 → 7일 이내 단축 (67% 단축)
2. AI 자동심사 적용률 연간 처리 건수의 85% 달성
3. 오심률 3.2% → 0.5% 이하 감소 (실시간 법령 기준 자동 대조)
4. 부정수급 자동 탐지율 기존 대비 350% 향상

## 기술제안서

### 적용 기술
- **OCR 엔진**: 한글 특화 딥러닝 OCR (문자 인식 정확도 99.2%, 손글씨 포함)
- **NLP 분석**: KoBERT 기반 문서 분류 + 정보 추출 (급여 유형·자격 요건 자동 파악)
- **이상 탐지**: GNN(그래프 신경망) 기반 부정수급 네트워크 탐지 (가족 관계 연쇄 분석)
- **추론 엔진**: 규칙 기반 + ML 하이브리드 (법령 개정 시 규칙 업데이트 30분 이내)

### 시스템 아키텍처
온프레미스 하이브리드 구조: 개인정보가 포함된 원문 데이터는 G-Cloud 보안 존에서 처리하고, 비식별화된 특징 벡터만 AI 분석 레이어로 전달한다. 12개 행정 DB는 사회보장정보원 표준 API를 통해 실시간 조회하며, 결과는 기존 복지업무관리시스템(행복e음) 화면에 임베드된다.

### 보안 대책
- CC인증 EAL4+ 획득 예정 (착수 후 6개월, 국가보안기술연구소 인증)
- 개인정보 처리 시스템 ISMS-P 인증 보유 (2024년 갱신)
- 역할 기반 접근 제어(RBAC), 담당 지역 외 조회 차단
- 모든 조회·처리 이력 감사 로그 10년 보관

### 차별성
1. **행복e음 완전 통합**: 별도 시스템 전환 불필요 → 현업 적응 기간 Zero
2. **실시간 법령 연동**: 보건복지부 법령정보 API 직접 연결, 기준 변경 30분 내 반영
3. **설명 가능 AI**: 심사 결정 근거를 자연어로 출력 → 이의신청 대응 자료 자동 생성

## 기대효과

### 정량적 효과
- **심사 기간**: 21일 → 7일 단축 / 연간 비용 절감 43억 원
- **오심 감소**: 오심률 3.2% → 0.5% / 오심 정정 비용 연 18억 원 절감
- **부정수급 차단**: 연간 63억 원 환수 증가 (탐지율 350% 향상 기반 추산)
- **처리 능력**: 일 3,900건 → 9,600건 (146% 증가)
- **시민 만족도**: 복지 서비스 만족도 현 61점 → 목표 83점 (22점 상승)

### 사회적 가치
위기 가구의 복지급여 수급 대기 기간이 3주에서 1주로 단축됨으로써, 긴급 생계 위기 상황에서 실질적 보호가 가능해진다. AI 판단 근거의 투명한 제공으로 복지 행정의 신뢰도를 제고하고, 담당 공무원의 단순 업무 부담 경감으로 서비스 품질이 향상된다.
