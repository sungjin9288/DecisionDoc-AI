## 1. 시작하기

### 1.1 로그인
1. 브라우저에서 서비스 URL 접속
2. 아이디와 비밀번호 입력 후 **로그인** 클릭
3. 기관 SSO 사용 시 **기관 계정으로 로그인** 클릭

> **최초 접속 시**: 관리자가 초대 링크를 발송합니다.
> 비밀번호를 잊은 경우 관리자에게 문의하세요.

### 1.2 화면 구성
| 영역 | 설명 |
|------|------|
| 상단 헤더 | 알림, 사용자 메뉴, 다크모드 전환 |
| 탭 네비게이션 | 문서 생성 / 결재함 / 프로젝트 / 스타일 / AI 성능 |
| 번들 그리드 | 생성할 문서 유형 선택 |
| 생성 폼 | 제목, 목표, 컨텍스트 입력 |

### 1.3 역할별 권한
| 역할 | 문서 생성 | 결재 참여 | 사용자 관리 |
|------|-----------|-----------|-------------|
| Admin | O | O | O |
| Member | O | O | X |
| Viewer | O | 읽기만 | X |

## 2. 문서 생성

### 2.1 번들 선택
1. **문서 생성** 탭 클릭
2. 번들 그리드에서 생성할 문서 유형 클릭
   - 카테고리 필터로 빠른 검색 가능
   - 즐겨찾기 등록 가능

**주요 번들**
| 번들 | 용도 |
|------|------|
| RFP 분석서 | 제안요청서 분석 및 수주 전략 |
| AI 사업 제안서 | 나라장터 공공 IT 제안서 전체 |
| 사업수행계획서 | 과업 착수 후 수행 계획 |
| 중간보고서 | 진척률 및 이슈 현황 보고 |
| 준공보고서 | 과업 완료 및 납품물 목록 |
| 과업지시서 | 발주처용 SOW 작성 |

### 2.2 내용 입력
1. **제목**: 문서/사업 명칭 입력 (필수)
2. **대상 독자**: 경영진/실무자/기술팀 중 선택
3. **문서 톤**: 격식체/전문적/중립적 중 선택
4. **목표**: 이 문서로 달성하고자 하는 것 (필수)
5. **컨텍스트**: 배경 정보, 관련 데이터
6. **제약조건**: 예산, 기간, 특별 요구사항

> **팁**: 컨텍스트가 구체적일수록 문서 품질이 높아집니다.
> 예산, 기간, 기관명, 수치 데이터를 포함하세요.

### 2.3 나라장터 공고 자동 입력
1. **나라장터 공고 불러오기** 패널 클릭
2. **URL / 공고번호** 탭에 공고 URL 또는 번호 입력
3. **불러오기** 클릭 → 자동으로 제목/목표/컨텍스트 채워짐

### 2.4 파일 첨부
지원 형식: **PDF, DOCX, HWP, XLSX, CSV, TXT**
1. **파일 선택** 또는 파일을 드래그앤드롭
2. **RFP 자동 분석** 클릭 → AI가 주요 정보 추출 후 자동 입력

### 2.5 생성 및 확인
1. **생성하기** 클릭 (또는 `Cmd+Enter`)
2. **스케치 단계**: AI가 문서 구조 제안
   - 섹션 제목/내용 직접 편집 가능
   - **이 구조로 생성하기** 클릭
3. **생성 완료**: 탭별로 각 문서 확인

### 2.6 다운로드
생성 완료 후 우측 버튼으로 원하는 형식 다운로드:
| 버튼 | 형식 | 비고 |
|------|------|------|
| Word | .docx | 편집 가능 |
| HWP | .hwpx | 한글 전용 |
| PDF | .pdf | 인쇄/제출용 |
| Excel | .xlsx | 표/계획 문서 |
| PPT | .pptx | 발표 자료 |

**공문서 양식 적용**: 공문서 양식 옵션 패널에서 문서번호, 수신, 결재란 등 설정

## 3. 나라장터 연동

### 3.1 공고 검색
1. 나라장터 패널 → **키워드 검색** 탭
2. 사업명 키워드 입력 (예: "AI 시스템 구축")
3. 검색 기간 선택 → **검색**
4. 결과 목록에서 공고 클릭 → 자동 입력

### 3.2 API 키 설정 (관리자)
**ops 패널** (`?ops=1`) → G2B API 키 입력
- 발급: [data.go.kr](https://data.go.kr) → "나라장터 입찰공고정보서비스"

> API 키 없이도 URL 직접 입력 방식으로 사용 가능

## 4. 결재 워크플로우

### 4.1 결재 요청 (기안자)
1. 문서 생성 완료 후 **결재 요청** 클릭
2. 기안자 이름, 검토자, 결재자 입력
3. **요청** 클릭

### 4.2 결재 처리
**결재함** 탭 → 이름 입력 → **조회**

| 상태 | 의미 | 다음 액션 |
|------|------|-----------|
| 기안 | 초안 작성됨 | 검토 요청 |
| 검토 중 | 검토자 확인 중 | 검토 완료/수정 요청 |
| 수정 요청 | 수정 필요 | 수정 후 재검토 요청 |
| 승인 | 결재 완료 | 문서 다운로드 |
| 반려 | 결재 거부 | 내용 수정 후 재기안 |

### 4.3 의견 작성
결재 상세 화면 하단 메시지 창에서 `@이름`으로 멘션 가능

## 5. 프로젝트 관리

### 5.1 프로젝트 생성
**프로젝트** 탭 → **+ 새 프로젝트**
- 프로젝트명, 발주처, 계약번호, 회계연도 입력

### 5.2 문서 연결
문서 생성 시 **프로젝트 선택** 드롭다운에서 프로젝트 선택
→ 생성된 문서가 자동으로 프로젝트에 추가됨

### 5.3 Voice Brief 요약 가져오기
기존 프로젝트 상세 화면에서 Voice Brief summary를 바로 가져올 수 있습니다.

1. **프로젝트** 탭에서 대상 프로젝트를 엽니다.
2. **Voice Brief 요약 가져오기** 카드에서 `Recording ID` 를 입력합니다.
3. 특정 summary revision을 고정할 때만 `Revision ID` 를 입력합니다.
4. **가져오기** 를 누르면 프로젝트에 Voice Brief 문서가 추가됩니다.

가져오기 후에는 프로젝트 문서에 아래 source 정보가 함께 저장됩니다.

- `source_recording_id`
- `source_summary_revision_id`
- `source_review_status`
- `source_sync_status`
- `source_use_case`
- `source_audio_url`

실패 시 표시되는 대표 상태:

| 코드 | 의미 |
|------|------|
| `stale_summary` | 최신 summary가 아니라서 재생성 또는 재동기화가 필요 |
| `unapproved_summary` | summary가 승인되지 않아 import 차단 |
| `voice_brief_not_found` | `recording_id` 또는 `revision_id` 가 upstream에 없음 |
| `voice_brief_upstream_error` | upstream 장애 또는 일시적 오류 |
| `voice_brief_not_configured` | 현재 환경에 Voice Brief 연동 설정이 없음 |

### 5.4 연도별 아카이브
프로젝트 상세 → **연도별 아카이브** 버튼

### 5.5 Public Procurement Go/No-Go Copilot
프로젝트 상세에서 공고 연결부터 판단, 문서 생성, 결재/공유까지 한 번에 진행할 수 있습니다.

1. **프로젝트** 탭에서 대상 프로젝트를 엽니다.
2. **Public Procurement Go/No-Go Copilot** 카드에서 `URL / 공고번호` 를 입력합니다.
3. **공고 가져오기** 를 누르면 공고가 프로젝트 procurement state에 연결됩니다.
4. **판단 갱신** 을 누르면 시스템이 아래를 순차적으로 계산합니다.
   - deterministic hard filter
   - weighted soft-fit score
   - recommendation: `GO` / `CONDITIONAL_GO` / `NO_GO`
   - bid-readiness checklist
5. 화면 하단 summary에서 아래 핵심 정보를 바로 확인할 수 있습니다.
   - 공고명 / 발주기관 / 예산 / 마감일
   - recommendation badge
   - soft-fit score
   - blocking hard-filter 건수
   - action-needed checklist 건수
   - missing data 요약

### 5.6 의사결정 문서와 downstream 문서 생성
판단이 끝나면 같은 카드에서 기존 문서 흐름으로 바로 넘어갈 수 있습니다.

- **의사결정 문서 생성**
  - `bid_decision_kr` 를 `POST /generate/stream` 경로로 생성합니다.
  - 생성된 문서는 현재 프로젝트 문서 목록에 자동 추가됩니다.
- **RFP 분석 / 제안서 / 수행계획**
  - recommendation 이 `GO` 또는 `CONDITIONAL_GO` 일 때만 활성화됩니다.
  - 각각 `rfp_analysis_kr`, `proposal_kr`, `performance_plan_kr` 로 연결됩니다.
  - 앞 단계에서 만든 structured procurement state가 downstream context로 재사용됩니다.

### 5.7 프로젝트 문서에서 결재 요청 / 공유
project detail 문서 목록에서 기존 workflow를 그대로 재사용할 수 있습니다.

- **결재 요청**
  - 문서 row의 **결재 요청** 을 누르면 기존 `/approvals` workflow로 연결됩니다.
  - 이미 결재가 생성된 문서는 **결재 보기** 로 바뀌며, 기존 결재 상세로 이동합니다.
- **공유**
  - 문서 row의 **공유** 를 누르면 기존 `/share` 링크가 생성됩니다.
- **다운로드**
  - PDF / Word / HWP 다운로드 버튼은 기존 동작을 그대로 사용합니다.

### 5.8 procurement 관련 동작 조건
- 이 기능은 운영 환경에서 `DECISIONDOC_PROCUREMENT_COPILOT_ENABLED=1` 일 때만 보입니다.
- `GO` / `CONDITIONAL_GO` 판단 전에는 downstream shortcut 이 비활성화됩니다.
- 공고번호 입력 방식은 운영 환경의 `G2B_API_KEY` 설정 여부에 따라 실패할 수 있습니다.
  URL 입력 방식은 스크래핑이 가능하면 API 키 없이도 사용할 수 있습니다.

### 5.9 v1.1.0 웹 검증 체크리스트
`v1.1.0` 배포 직후에는 아래 순서로 실제 사용자 흐름을 한 번 확인하는 것을 권장합니다.

1. 운영 URL에 접속하고 로그인합니다.
2. 기존 프로젝트를 열거나 새 프로젝트를 하나 만듭니다.
3. 프로젝트 상세에서 **Public Procurement Go/No-Go Copilot** 카드가 보이는지 확인합니다.
4. `URL / 공고번호` 입력란에 `R26BK01398367` 를 넣고 **공고 가져오기** 를 누릅니다.
5. summary 영역에 공고명, 발주기관, recommendation, score 요약이 채워지는지 확인합니다.
6. **판단 갱신** 을 눌러 recommendation badge 와 checklist 요약이 갱신되는지 확인합니다.
7. **의사결정 문서 생성** 을 눌러 `bid_decision_kr` 문서가 프로젝트 문서 목록에 추가되는지 확인합니다.
8. 생성된 문서 row에 **결재 요청** 과 **공유** 버튼이 모두 보이는지 확인합니다.
9. 필요하면 `/version` 엔드포인트를 열어 아래 값이 맞는지 확인합니다.
   - `version: 1.1.0`
   - `features.procurement_copilot: true`

실패 시 함께 확인하면 좋은 항목:

- 브라우저 console error
- 실패한 network request path 와 status code
- import 에 사용한 입력값이 bid number 인지, detail URL 인지
- 프로젝트가 비어 있는 새 프로젝트인지, 기존 문서가 많은 프로젝트인지

## 6. 스타일 설정

### 6.1 프로필 생성
**스타일** 탭 → **+ 새 스타일 프로필**

### 6.2 톤 가이드 설정
| 항목 | 옵션 |
|------|------|
| 문체 | 경어체/해요체/합쇼체/혼용 |
| 서술 밀도 | 간결하게/보통/상세하게 |
| 서술 관점 | 1인칭/3인칭/기관명칭/혼용 |
| 추가 규칙 | 직접 작성 |
| 금지 표현 | 쉼표로 구분 |

### 6.3 문서에서 스타일 학습
**문서 업로드** → 기존 문서(PDF/DOCX/HWP) 업로드
→ AI가 문체 자동 분석 후 프로필에 반영

### 6.4 기본 프로필 설정
**기본으로 설정** → 이후 생성되는 모든 문서에 자동 적용

## 7. 관리자 기능

(`?ops=1` URL 파라미터로 접근)

### 7.1 사용자 관리
**관리자 → 사용자 목록**
- 신규 사용자 초대
- 역할 변경 (admin/member/viewer)
- 계정 비활성화

### 7.2 SSO 설정
**ops 패널 → SSO 설정**
- LDAP/AD: 기관 내부망 계정 연동
- SAML 2.0: 기관 SSO 연동
- G-Cloud IAM: 정부통합전산센터 연동

### 7.3 감사 로그
**ops 패널 → 보안 감사 로그**
- 로그인 실패, 문서 다운로드, 결재 처리 등 모든 행위 기록
- CSV 내보내기 지원
- 의심 IP 자동 탐지

### 7.4 요금제 관리
**ops 패널 → 요금제**
- 현재 사용량 확인
- 플랜 업그레이드

## 8. 문제 해결

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| 로그인 실패 반복 | Rate limit 적용 | 15분 후 재시도 |
| 문서 생성 지연 | LLM API 응답 지연 | 취소 후 재시도 |
| HWP 열리지 않음 | 한글 2010 이상 필요 | 한글 최신 버전 설치 |
| PDF 텍스트 추출 실패 | 스캔 이미지 PDF | DOCX로 변환 후 업로드 |
| 생성 오류 (402) | 사용량 한도 초과 | 관리자에게 플랜 업그레이드 요청 |
| SSO 로그인 안 됨 | 기관 SSO 설정 오류 | 관리자에게 LDAP/SAML 설정 확인 요청 |
