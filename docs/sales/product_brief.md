# DecisionDoc AI 제품 소개서

## 1. 제품 한 줄 설명

DecisionDoc AI는 조직이 실제 업무 문서를 더 빠르게 만들고, 검토하고, 승인하고, 제출 가능한 형식으로 내보낼 수 있게 하는 내부 설치형 AI 문서 운영 플랫폼입니다.

## 2. 해결하는 문제

많은 조직은 아래 문제를 동시에 겪습니다.

- 문서 초안 작성에 시간이 많이 든다.
- 자료는 많지만, 실제 제출 가능한 문서로 정리하는 데 인력이 소모된다.
- 문서 생성 후 검토, 결재, 공유, 보관 흐름이 분리되어 있다.
- 공공조달, 제안서, 수행계획서 같은 업무 문서는 형식과 품질 기준이 높다.
- 외부 SaaS에 민감한 자료를 넣기 어렵다.

DecisionDoc AI는 이 문제를 문서 생성 기능 하나로 해결하지 않고, 프로젝트 중심 워크플로우로 묶어서 해결합니다.

## 3. 핵심 가치

### 3.1 문서 생성에서 끝나지 않는다

DecisionDoc AI는 단순한 AI 작성 도구가 아닙니다.
문서 생성 이후의 검토, 결재 요청, 공유, 이력 관리, 다운로드까지 같은 플랫폼에서 처리합니다.

### 3.2 조직 문서에 맞는 산출물을 만든다

대화 요약이 아니라 실제 업무 산출물을 목표로 합니다.
DOCX, PDF, HWP, XLSX, PPTX, ZIP 등 제출/배포 가능한 형식으로 결과물을 제공합니다.

### 3.3 내부 설치형 운영이 가능하다

Docker Compose 기반으로 사내 서버나 온프레미스 환경에 설치할 수 있습니다.
고객사별 분리 운영, 키 분리, 데이터 분리, 감사 로그 운영이 가능합니다.

### 3.4 공공조달 업무에 특화된 확장성이 있다

나라장터 공고 조회, Go/No-Go 판단, RFP 분석, 제안서 및 수행계획 문서 생성 흐름을 지원합니다.

## 4. 주요 기능

- AI-assisted document generation
- project/workspace management
- document collaboration, review, approval, sharing, and history
- knowledge document ingestion and reuse
- public procurement / G2B support
- export flows: DOCX, PDF, HWP, XLSX, PPTX, ZIP
- evaluation and operations tooling
- audit logging and role-based access control

## 5. 적합한 고객

- 제안서, 수행계획서, 보고서 작성 비중이 높은 조직
- 공공입찰/공공조달 업무를 수행하는 기업
- 외부 SaaS보다 내부 설치와 데이터 통제를 선호하는 조직
- 여러 법인/장소/고객사 환경을 분리 운영해야 하는 조직

## 6. 도입 효과

- 초안 작성 시간 단축
- 문서 형식 표준화
- 프로젝트별 문서 이력 관리 강화
- 결재/공유 흐름 일원화
- 공공조달 대응 속도 향상
- 내부 운영과 감사 대응 용이성 확보

## 7. 도입 방식

### 기본 권장안

- 배포: Docker Compose
- 운영: 내부 설치형
- 환경: 고객사별 독립 배포
- 접근: 웹 브라우저

### 운영 확장안

- 운영자 공용 환경과 고객 전용 환경을 독립 운영
- 환경별 도메인, API 키, 운영 키, 데이터 볼륨 분리
- 동일 버전 릴리즈를 순차 배포

## 8. 한 줄 포지셔닝

DecisionDoc AI는 자료를 요약해 주는 도구가 아니라, 조직이 실제 문서를 만들고 승인하고 제출하도록 돕는 문서 운영 플랫폼입니다.
