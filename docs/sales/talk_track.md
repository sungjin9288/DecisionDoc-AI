# DecisionDoc AI 대표 미팅 Talk Track

## 1. 사용 목적

이 문서는 외부 대표, 임원, 의사결정권자 앞에서 DecisionDoc AI를 설명할 때 바로 읽거나 참고할 수 있도록 만든 운영자용 말하기 스크립트입니다.

상황에 따라 아래 3개 길이 중 하나를 선택해서 사용합니다.

- 30초 버전: 첫 인사 직후 제품 포지셔닝만 전달할 때
- 2분 버전: 제품 개요와 차별점을 짧게 설명할 때
- 5분 버전: 시연 전후로 도입 방식과 운영 구조까지 같이 설명할 때

## 2. 30초 버전

`DecisionDoc AI는 단순히 자료를 읽고 정리하는 도구가 아니라, 조직이 실제 문서를 만들고 검토하고 승인하고 제출하는 내부 설치형 AI 문서 운영 플랫폼입니다.`

`즉, 대화형 답변에서 끝나는 것이 아니라 실제 업무 산출물과 운영 흐름까지 연결하는 것이 핵심입니다.`

## 3. 2분 버전

`많은 조직은 문서 초안 작성에 시간이 많이 들고, 자료는 많지만 실제 제출 가능한 형태로 정리하는 데 인력이 많이 들어갑니다.`

`DecisionDoc AI는 이 문제를 단순 생성 기능 하나로 해결하지 않고, 문서 생성부터 검토, 승인, 공유, 이력 관리, export까지 이어지는 운영형 구조로 해결하려고 만든 제품입니다.`

`NotebookLM 같은 도구가 읽기와 이해에 강하다면, DecisionDoc AI는 작성과 운영에 강합니다.`

`특히 내부 설치형 구조로 고객사별 환경 분리, 키 분리, 데이터 분리 운영이 가능하고, DOCX, PDF, HWP, XLSX, PPTX 같은 실제 제출형 산출물 중심으로 설계되어 있다는 점이 차별점입니다.`

## 4. 5분 버전

### 4.1 문제 정의

`실무에서는 문서를 잘 쓰는 것보다, 일정 안에 반복 가능한 품질로 만들고 조직 안에서 승인과 공유까지 끝내는 것이 더 어렵습니다.`

`특히 제안서, 수행계획서, 보고서, 공공조달 문서처럼 형식 요구가 높은 문서는 단순 AI 답변만으로는 부족합니다.`

### 4.2 제품 설명

`DecisionDoc AI는 조직이 실제 업무 문서를 더 빠르게 만들고, 검토하고, 승인하고, 제출 가능한 형식으로 내보내게 돕는 내부 설치형 AI 문서 운영 플랫폼입니다.`

`핵심은 단순 요약이나 질의응답이 아니라, project + knowledge + generation + approval/share + export + audit 흐름을 하나의 운영 구조로 묶는 데 있습니다.`

### 4.3 차별점

`첫째, 실제 문서 산출물 중심입니다. 결과가 단순 텍스트가 아니라 DOCX, PDF, HWP, XLSX, PPTX 같은 업무 파일로 이어집니다.`

`둘째, 내부 설치형으로 운영할 수 있습니다. 즉 고객사별 전용 환경, 도메인, 키, 데이터, 로그를 분리해서 운영할 수 있습니다.`

`셋째, 공공조달과 제안 업무처럼 구조화된 문서 운영에 맞춘 확장성이 있습니다.`

### 4.4 현재 보여줄 수 있는 범위

`현재는 admin.decisiondoc.kr 기준으로 HTTPS 접속, 문서 생성, 인증 제어, export 흐름까지 검증된 상태입니다.`

`지금 보시는 환경은 공용 데모 및 운영 환경이고, 실제 도입 시에는 고객사 전용 환경으로 별도 분리 배포하는 방식을 권장합니다.`

### 4.5 마무리 문구

`정리하면, DecisionDoc AI는 자료를 이해하는 도구가 아니라 조직이 실제 문서를 만들고 운영하는 플랫폼입니다.`

`만약 귀사에서 문서 생성 이후 승인, 공유, 제출, 데이터 통제까지 같이 중요하다면, 일반 요약형 AI보다 더 직접적인 운영 가치를 드릴 수 있습니다.`

## 5. 질문이 들어왔을 때 짧은 답변

### 이거 SaaS인가요?

`기본 권장 구조는 내부 설치형입니다. 웹으로 접속하지만 고객사별 독립 배포를 전제로 설명드리고 있습니다.`

### 우리 회사 데이터가 다른 고객과 섞이나요?

`실도입 시에는 귀사 전용 환경으로 분리합니다. 도메인, 키, 데이터, 운영 로그를 별도 관리하는 구조입니다.`

### 일반 AI 툴과 가장 다른 점은 뭔가요?

`DecisionDoc AI는 답변 생성보다 실제 문서 운영 흐름에 초점을 둔다는 점이 가장 큰 차이입니다.`

### 바로 도입 가능한가요?

`가능합니다. 현재 데모 환경은 공용 운영 환경이고, 실제 도입 시에는 귀사 전용 환경으로 분리해 배포하는 방식을 권장합니다.`

## 6. 사용 팁

- 첫 미팅에서는 30초 또는 2분 버전을 먼저 사용합니다.
- 상대가 설치, 보안, 데이터 분리를 묻기 시작하면 5분 버전으로 확장합니다.
- 기능을 모두 보여주기보다 “문서 생성 → 운영 → 제출” 흐름만 강조하는 쪽이 설득력이 높습니다.
