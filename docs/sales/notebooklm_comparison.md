# DecisionDoc AI vs NotebookLM

## 1. 비교 요약

NotebookLM은 자료를 읽고 이해하고 요약하는 데 강한 AI research assistant에 가깝습니다.
DecisionDoc AI는 조직이 실제 문서를 생성하고, 검토하고, 승인하고, 내보내는 문서 운영 플랫폼에 가깝습니다.

## 2. 차이점

| 항목 | NotebookLM | DecisionDoc AI |
|------|------------|----------------|
| 기본 목적 | 자료 이해, 요약, 대화, 인용 | 조직 문서 생성, 검토, 승인, 제출 |
| 산출물 | 요약, 노트, 오디오 오버뷰, 학습용 결과 | DOCX, PDF, HWP, XLSX, PPTX 등 업무 산출물 |
| 프로젝트 관리 | 제한적 | 프로젝트/워크스페이스 중심 운영 |
| 결재/승인 | 없음 | 있음 |
| 공유/이력 | 제한적 | 문서 이력, 결재 흐름, 공유 지원 |
| 내부 설치 | 일반적으로 SaaS 중심 | Docker 기반 내부 설치 가능 |
| 고객사 분리 운영 | 제한적 | 고객사/장소별 독립 운영 가능 |
| 공공조달 특화 | 없음 | G2B, Go/No-Go, RFP downstream 지원 |

## 3. NotebookLM이 더 잘 맞는 경우

- 개인 또는 소규모 팀이 자료를 빠르게 읽고 이해하려는 경우
- PDF, 웹자료, 회의자료를 넣고 즉시 요약/질의응답을 하고 싶은 경우
- 제출용 문서보다 리서치 보조가 더 중요한 경우

## 4. DecisionDoc AI가 더 잘 맞는 경우

- 최종 산출물이 실제 문서여야 하는 경우
- 결재, 승인, 공유, 보관 흐름까지 같이 관리해야 하는 경우
- 고객사 또는 장소별로 환경을 분리 운영해야 하는 경우
- 내부망 또는 사내 설치형 운영이 필요한 경우
- 공공조달/제안/수행계획 문서가 핵심 업무인 경우

## 5. 우리 제품의 강점

### 5.1 업무 완결성

문서 초안만 만드는 것이 아니라, 생성 이후의 승인과 운영까지 이어집니다.

### 5.2 제출 가능한 산출물

대화형 결과가 아니라 실제 제출 가능한 파일 형식으로 내보낼 수 있습니다.

### 5.3 내부 통제

환경별 키 분리, 데이터 분리, 감사 로그, 역할 기반 접근 제어가 가능합니다.

### 5.4 공공조달 특화

일반 지식 요약이 아니라 공고 기반 판단과 문서 생산 흐름까지 제공합니다.

## 6. 판매 시 설명 문구

### 짧은 설명

NotebookLM은 자료를 이해하는 도구이고, DecisionDoc AI는 조직이 실제 의사결정 문서를 만들고 승인하고 제출하는 플랫폼입니다.

### 고객 설명용 문구

NotebookLM이 “읽기와 이해”에 강하다면, DecisionDoc AI는 “작성과 운영”에 강합니다.
특히 내부 설치, 프로젝트 관리, 결재, 공공조달 문서화처럼 조직 업무 흐름이 중요한 환경에서 차이가 분명합니다.
