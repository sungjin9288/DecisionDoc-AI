# DecisionDoc AI 소개서

## 1. 제품 개요

DecisionDoc AI는 조직이 실제 업무 문서를 더 빠르게 만들고, 검토하고, 승인하고, 제출 가능한 형식으로 내보낼 수 있게 돕는 내부 설치형 AI 문서 운영 플랫폼입니다.

이 제품은 단순한 요약 도구가 아니라, 문서 초안 작성부터 검토, 승인, 공유, 보관, 제출 포맷 변환까지 이어지는 실제 운영 흐름을 하나의 웹 기반 환경으로 묶는 것을 목표로 합니다.

## 2. 어떤 문제를 해결하는가

- 문서 초안 작성에 시간이 오래 걸립니다.
- 자료는 많지만 제출 가능한 형태로 정리하는 데 인력이 많이 듭니다.
- 제안서, 수행계획서, 보고서 같은 문서는 형식과 품질 요구 수준이 높습니다.
- 문서 생성 이후 검토, 결재, 공유, 이력 관리가 분산되어 있습니다.
- 민감한 자료를 외부 SaaS에 그대로 넣기 어려운 조직이 많습니다.

DecisionDoc AI는 위 문제를 단순 생성 기능이 아니라 프로젝트 중심 운영 워크플로우로 해결합니다.

## 3. 핵심 가치

### 3.1 문서 생성에서 끝나지 않는 운영형 구조

DecisionDoc AI는 생성 이후의 검토, 승인 요청, 공유, 기록, 다운로드까지 같은 플랫폼 안에서 다룰 수 있도록 설계되어 있습니다.

### 3.2 실제 제출 가능한 산출물 중심

대화 요약이 아니라 실제 업무에 제출하거나 배포할 수 있는 문서를 목표로 합니다.

- DOCX
- PDF
- HWP
- XLSX
- PPTX
- ZIP 패키지

### 3.3 내부 설치형 배포 가능

Docker Compose 기반으로 사내 서버, 온프레미스, AWS VM 환경에 배포할 수 있습니다.

- 고객사별 환경 분리
- 도메인 분리
- API 키 분리
- 운영 키 분리
- 데이터 볼륨 분리
- 로그 및 감사 대응 체계 확보

### 3.4 공공조달 대응 확장성

나라장터 공고 조회, Go or No-Go 판단, RFP 분석, 제안서 및 수행계획 문서 생성 흐름까지 확장할 수 있습니다.

## 4. 주요 기능

- AI 기반 문서 생성
- 프로젝트 및 작업 단위 관리
- 문서 검토, 승인, 공유, 이력 추적
- 지식 문서 적재 및 재활용
- 공공조달 및 G2B 대응 흐름
- DOCX, PDF, HWP, XLSX, PPTX, ZIP 내보내기
- 평가 및 운영 지원 기능
- 감사 로그 및 권한 기반 운영

## 5. 적합한 고객

- 제안서, 수행계획서, 보고서 작성 비중이 높은 조직
- 공공입찰 또는 공공조달 업무를 수행하는 기업
- 내부 설치와 데이터 통제를 선호하는 조직
- 여러 법인, 장소, 고객사 환경을 분리 운영해야 하는 조직

## 6. 경쟁 제품과의 차이

일반적인 생성형 AI 도구나 NotebookLM 계열 제품은 자료를 읽고 정리하는 데 강점이 있지만, 실제 조직 문서 운영까지 연결되는 경우는 제한적입니다.

DecisionDoc AI의 차별점은 다음과 같습니다.

- 실제 업무 문서 작성 흐름에 초점을 맞춘 구조
- 승인, 공유, 이력, 내보내기까지 포함한 운영형 플랫폼
- 내부 설치형 배포와 고객사별 분리 운영 가능
- 공공조달과 제안서 업무에 맞춘 확장 가능성

## 7. 도입 방식

### 기본 권장안

- 배포 방식: Docker Compose
- 운영 형태: 내부 설치형
- 접근 방식: 웹 브라우저
- 환경 구성: 운영자 공용 환경 + 고객 전용 환경 분리

### 운영 예시

- 공용 운영 및 데모 환경: admin 서브도메인
- 고객사 전용 환경: 고객사별 독립 서브도메인

이 구조를 사용하면 소개, 데모, 파일럿 검토는 공용 환경에서 진행하고, 실제 도입 시에는 고객 전용 환경으로 분리 전환할 수 있습니다.

## 8. 기대 효과

- 초안 작성 시간 단축
- 문서 형식 표준화
- 검토 및 승인 흐름 일원화
- 프로젝트별 이력 관리 강화
- 공공조달 대응 속도 향상
- 내부 운영과 감사 대응 용이성 확보

## 9. 요약

DecisionDoc AI는 자료를 요약해 주는 도구가 아니라, 조직이 실제 문서를 만들고 승인하고 제출하도록 돕는 내부 설치형 문서 운영 플랫폼입니다.
