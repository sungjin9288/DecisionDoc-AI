# DecisionDoc AI 미팅용 1장 요약

## 1. 한 줄 설명

DecisionDoc AI는 조직이 실제 업무 문서를 더 빠르게 만들고, 검토하고, 승인하고, 제출 가능한 형식으로 내보내도록 돕는 내부 설치형 AI 문서 운영 플랫폼입니다.

## 2. 왜 필요한가

- 문서 초안 작성에 시간이 많이 듭니다.
- 자료는 많지만 제출 가능한 문서로 정리하는 데 인력이 소모됩니다.
- 문서 생성 후 검토, 승인, 공유, 보관 흐름이 분산되어 있습니다.
- 외부 SaaS에 민감 문서를 그대로 넣기 어려운 조직이 많습니다.

## 3. 핵심 차별점

- 단순 요약이 아니라 실제 업무 문서 생성과 운영에 초점을 둡니다.
- DOCX, PDF, HWP, XLSX, PPTX, ZIP 같은 제출형 산출물을 지원합니다.
- 내부 설치형으로 고객사별 도메인, 키, 데이터, 볼륨을 분리 운영할 수 있습니다.
- 공공조달, 제안서, 수행계획 문서 흐름까지 확장 가능합니다.

## 4. 지금 바로 보여줄 수 있는 것

- 웹 접속: `https://admin.decisiondoc.kr`
- 문서 생성 요청 및 결과 확인
- 인증 없는 요청 차단과 인증 요청 성공 흐름
- export 결과 생성

## 5. 도입 시 운영 방식

- 현재 공용 데모/운영 환경: `admin.decisiondoc.kr`
- 실제 도입 시: 고객사 전용 서브도메인과 전용 배포 환경 분리
- 권장 배포: Docker Compose 기반 내부 설치형

## 6. 비교 포인트

NotebookLM이 자료를 읽고 이해하는 도구라면, DecisionDoc AI는 조직이 실제 문서를 만들고 운영하는 플랫폼입니다.

## 7. 기대 효과

- 초안 작성 시간 단축
- 문서 형식 표준화
- 승인과 공유 흐름 일원화
- 고객사별 데이터 통제 강화

## 8. 후속 제안

- 공용 데모 후 고객사 전용 환경 분리 배포
- 파일럿 범위 정의
- 실제 문서 양식과 운영 정책에 맞춘 커스터마이징 협의
